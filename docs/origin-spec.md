# 需求規格書 (Origin Spec)

## 一、任務目標

建立一個 Full-stack Web 系統，使用者輸入「台股股票代號」後，顯示該股票最近 30 個交易日的相關資訊。

---

## 二、功能需求

### 2.1 基本功能

- 使用者輸入台股股票代號（如 `2330`）
- 取得該股票「最近 30 個交易日」的交易資料
- 計算技術指標：
  - **MA5**（5 日移動平均線）
  - **MA20**（20 日移動平均線）

### 2.2 前端顯示

- 股價走勢圖（以收盤價為主）
- MA5 / MA20 均線曲線疊加於走勢圖上

### 2.3 AI 分析（繁體中文）

- 僅根據系統中實際顯示的數據進行描述
- 至少引用 2 項具體數據事實（例如均線交叉、價格變化幅度）
- 不可提供投資建議、預測或主觀推薦
- 不可使用無法由資料直接支持的推論

### 2.4 資料可信度提示

當以下情況發生時，系統應明確提示使用者，且不輸出可能誤導的結果：

- 資料筆數不足（例如不足 30 個交易日）
- 資料缺漏或異常（例如欄位為空字串）
- 指標無法計算（例如不足 5 筆無法算 MA5、不足 20 筆無法算 MA20）
- AI 分析缺乏足夠依據

## 三、本地資料盤點

專案目錄下現有 4 個 JSON 檔案：

### 3.1 STOCK_DAY_ALL.json — 個股每日交易資料

| 欄位 | 說明 | 範例 |
|---|---|---|
| Date | 民國曆日期 | `1150402`（= 2026/04/02） |
| Code | 股票代號 | `0050` |
| Name | 股票名稱 | `元大台灣50` |
| TradeVolume | 成交股數 | `113976137` |
| TradeValue | 成交金額 | `8478357215` |
| OpeningPrice | 開盤價 | `75.90` |
| HighestPrice | 最高價 | `76.05` |
| LowestPrice | 最低價 | `73.80` |
| ClosingPrice | 收盤價 | `73.95` |
| Change | 漲跌價差 | `-1.5000` |
| Transaction | 成交筆數 | `162558` |

### 3.2 STOCK_DAY_AVG_ALL.json — 收盤價與月均價

| 欄位 | 說明 |
|---|---|
| Date | 民國曆日期 |
| Code | 股票代號 |
| Name | 股票名稱 |
| ClosingPrice | 收盤價 |
| MonthlyAveragePrice | 月平均價 |

### 3.3 BWIBBU_ALL.json — 本益比、殖利率、淨值比

| 欄位 | 說明 |
|---|---|
| Date | 民國曆日期 |
| Code | 股票代號 |
| Name | 股票名稱 |
| PEratio | 本益比（可能為空） |
| DividendYield | 殖利率 |
| PBratio | 股價淨值比 |

### 3.4 holidaySchedule.json — 休市日程表

| 欄位 | 說明 |
|---|---|
| Name | 假日名稱 |
| Date | 民國曆日期 |
| Weekday | 星期 |
| Description | 說明 |

### 3.5 本地資料限制

- **僅包含 1 個交易日**（1150402 = 2026/04/02），共 1348 支股票
- **無法單靠本地資料完成需求**：30 個交易日需從外部 API 取得歷史資料
- 部分股票欄位值為空字串，需做防呆處理
- 日期格式為民國曆，需轉換處理

## 四、已辨識的模糊處與風險

### 4.1 模糊處

| 項目 | 說明 | 決策 |
|---|---|---|
| 「最近 30 個交易日」定義 | 是以今天為基準往回推 30 個「有開市」的日子，還是日曆天？ | 採用 30 個實際交易日（排除假日與週末） |
| 資料來源 | 題目未指定使用哪個 API | **以 TWSE 傳統網站 API 作為主要歷史資料來源**（詳見第六章），TWSE OpenAPI / 本地 JSON 作為當日補充 |
| AI 分析使用的模型 | 未限定使用哪種 AI 模型 | 可使用 OpenAI / Claude API，或後端以規則引擎產生分析文字 |
| 股價走勢圖類型 | 未指定 K 線圖或折線圖 | 預設使用收盤價折線圖，清楚呈現趨勢 |

### 4.2 風險點

| 風險 | 影響 | 緩解策略 |
|---|---|---|
| TWSE 傳統 API 可能有頻率限制（已知會擋高頻請求） | 連續查詢被阻擋 | 查詢間加入適當延遲（如 0.5~1 秒）；加入錯誤處理與使用者提示 |
| 每次查詢僅回傳一整個月，取 30 個交易日需查 2~3 個月 | 回應時間較長 | 後端合併多月資料後截取最近 30 筆；可加入快取機制 |
| 新上市股可能交易日不足 30 天 | MA20 無法計算 | 明確提示「資料不足，僅顯示可計算範圍」，不輸出誤導結果 |
| AI 分析可能產生不準確描述 | 誤導使用者 | 僅以結構化數據為輸入，限制 prompt 範圍，標註「AI 生成內容」 |
| 民國曆日期格式需轉換（`114/04/01`） | 日期處理錯誤 | 後端統一轉為西元日期，前端再依需求格式化顯示 |

## 五、合理假設

1. 使用者輸入的股票代號為台灣證券交易所的上市股票代號
2. 「最近 30 個交易日」以查詢當日為基準，往前計算 30 個有交易的日子
3. 本地 JSON 資料為 TWSE 公開資料格式的快照，可作為當日資料補充來源
4. holidaySchedule.json 可用於輔助判斷交易日
5. 系統不需要即時報價功能，僅需歷史收盤資料
6. 以 TWSE 傳統網站 API 作為 30 個交易日歷史資料的主要來源

---

## 六、資料來源策略與 API 取用方式

### 6.1 資料來源總覽

| 來源 | 用途 | 涵蓋範圍 |
|---|---|---|
| **TWSE 傳統網站 API（主要）** | 取得指定股票的歷史每日交易資料 | 以月為單位，可查詢任意歷史月份的個股日線 |
| **TWSE OpenAPI（補充）** | 取得當日全市場快照（本益比、殖利率等） | 當日所有上市股票 |
| **本地 JSON（補充）** | 與 TWSE OpenAPI 結構相同，作為離線/備援資料 | 僅 2026/04/02 當日 |

### 6.2 TWSE 傳統網站 API — 歷史資料取用方式（主要來源）

**端點**

```
GET https://www.twse.com.tw/exchangeReport/STOCK_DAY
    ?response=json
    &date={YYYYMMDD}
    &stockNo={股票代號}
```

**參數說明**

| 參數 | 說明 | 範例 |
|---|---|---|
| `response` | 回傳格式，固定填 `json` | `json` |
| `date` | 查詢月份（西元），該月任一天皆可 | `20260401`（查 2026 年 4 月） |
| `stockNo` | 台股股票代號 | `2330` |

**回傳格式**

```json
{
  "stat": "OK",
  "date": "20260401",
  "title": "115年04月 2330 台積電各日成交資訊",
  "fields": ["日期", "成交股數", "成交金額", "開盤價", "最高價", "最低價", "收盤價", "漲跌價差", "成交筆數"],
  "data": [
    ["115/04/01", "47,389,975", "87,770,917,498", "1,870.00", "1,870.00", "1,840.00", "1,855.00", "+15.00", "49,053"],
    ["115/04/02", "39,404,091", "72,445,375,498", "1,865.00", "1,865.00", "1,825.00", "1,840.00", "-15.00", "41,393"]
  ],
  "notes": ["..."],
  "total": 2
}
```

**欄位對照（data 陣列索引）**

| 索引 | TWSE 欄位名 | 對應意義 | 格式 |
|---|---|---|---|
| 0 | 日期 | 交易日期 | 民國曆 `115/04/01` |
| 1 | 成交股數 | 成交量 | 字串含逗號 `47,389,975` |
| 2 | 成交金額 | 成交總金額 | 字串含逗號 |
| 3 | 開盤價 | 開盤價 | 字串含逗號 `1,870.00` |
| 4 | 最高價 | 最高價 | 字串含逗號 |
| 5 | 最低價 | 最低價 | 字串含逗號 |
| 6 | 收盤價 | 收盤價 | 字串含逗號 |
| 7 | 漲跌價差 | 漲跌幅 | 字串含正負號 `+15.00` |
| 8 | 成交筆數 | 成交筆數 | 字串含逗號 |

**取用方式**

```python
import requests

def fetch_month(stock_no: str, date: str) -> list:
    """取得指定股票某月的每日交易資料。
    date 格式：YYYYMMDD（西元），如 '20260401'
    """
    url = "https://www.twse.com.tw/exchangeReport/STOCK_DAY"
    params = {"response": "json", "date": date, "stockNo": stock_no}
    resp = requests.get(url, params=params)
    result = resp.json()

    if result.get("stat") != "OK" or "data" not in result:
        return []

    return result["data"]


# 取得最近 30 個交易日：查詢當月 + 前 1~2 個月，合併後取最近 30 筆
import time
from datetime import datetime

def fetch_recent_30_days(stock_no: str) -> list:
    all_data = []
    now = datetime.now()

    for month_offset in range(0, 3):  # 當月、前1月、前2月
        year = now.year
        month = now.month - month_offset
        if month <= 0:
            month += 12
            year -= 1
        date_str = f"{year}{month:02d}01"
        data = fetch_month(stock_no, date_str)
        all_data = data + all_data  # 舊的放前面
        if month_offset < 2:
            time.sleep(0.5)  # 避免被 TWSE 擋請求

    # 取最近 30 筆
    return all_data[-30:] if len(all_data) >= 30 else all_data
```

**資料清洗注意事項**
- 日期需從民國曆轉西元：`115/04/01` → `2026/04/01`
- 數值欄位為字串且含逗號，需移除逗號後轉為數字：`"1,870.00"` → `1870.00`
- 漲跌價差含正負號：`"+15.00"` / `"-15.00"`
- 無效代號回傳 `stat: "很抱歉，沒有符合條件的資料!"`，無 `data` 欄位

**實測結果**（2026/04/07 測試）

| 查詢參數 | 結果 |
|---|---|
| `date=20260401&stockNo=2330` | 2 筆（115年4月至今） |
| `date=20260301&stockNo=2330` | 約 21 筆（115年3月整月） |
| `date=20260201&stockNo=2330` | 約 17 筆（115年2月整月） |
| 三個月合併 | 約 40 筆，取最近 30 筆即可滿足需求 |
| `stockNo=9999`（無效代號） | stat 回傳錯誤訊息，無 data |

**頻率限制**
- TWSE 傳統 API 已知會擋高頻請求
- 建議每次查詢間隔 0.5～1 秒
- 取 30 個交易日最多查 3 個月，共 3 次請求

### 6.3 TWSE OpenAPI — 當日快照取用方式（補充來源）

**Base URL**

```
https://openapi.twse.com.tw/v1
```

**可用端點**

| 端點 | 說明 | 回傳 |
|---|---|---|
| `GET /exchangeReport/STOCK_DAY_ALL` | 上市個股日成交資訊 | 當日所有股票的開高低收、成交量等 |
| `GET /exchangeReport/STOCK_DAY_AVG_ALL` | 日收盤價及月平均價 | 當日所有股票的收盤價與月均價 |
| `GET /exchangeReport/BWIBBU_ALL` | 本益比、殖利率、淨值比 | 當日所有股票的基本面指標 |

**取用方式**

```python
import requests

# 取得當日所有股票成交資訊
resp = requests.get("https://openapi.twse.com.tw/v1/exchangeReport/STOCK_DAY_ALL")
data = resp.json()  # List[dict]，結構同本地 STOCK_DAY_ALL.json

# 篩選特定股票
stock_2330 = [d for d in data if d["Code"] == "2330"]
```

**限制**
- 無法帶參數查詢歷史日期，僅回傳當日資料
- 無法篩選單一股票，一律回傳全市場（約 1348 筆）
- 日期格式為民國曆（`1150402` = 2026/04/02），需轉換

### 6.4 資料整合策略

```
使用者輸入股票代號
       │
       ▼
  TWSE 傳統 API 查詢當月 + 前 1~2 個月（間隔 0.5 秒）
       │
       ├─ stat: "OK" → 合併資料 → 取最近 30 筆 → 計算 MA5/MA20 → 前端顯示
       │
       └─ stat: 錯誤訊息 → 顯示「查無此股票」或「資料來源暫時無法連線」
       
  TWSE OpenAPI / 本地 JSON（補充）
       │
       └─ 提供當日本益比、殖利率等額外資訊（非必要，作為加值資料）
```

**原則：歷史股價計算（MA5/MA20、走勢圖）統一使用 TWSE 傳統 API 單一來源，資料為 TWSE 官方原始數據，避免第三方轉手造成數值差異。**
